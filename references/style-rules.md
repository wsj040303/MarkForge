# Style Rules

本文件用于将正文统一为正式书面语，适用于中文 Markdown 文档。

## 一、正文语气

正文默认采用：

- 说明文语气
- 讲义语气
- 项目文档 / 方案文档语气

避免采用：

- 聊天语气
- 口头讲解语气
- 问答式引导语气
- 经验分享式第一人称语气

## 二、应避免的表达

以下表达若出现在正文中，通常需要改写：

- 你、我们、大家、同学
- 先说结论
- 如果只记一句话
- 通俗一点
- 可以理解为
- 可以把它想象成
- 直接
- 快速
- 跑起来
- 其实
- 最好
- 学会

说明：

- 若上述表达出现在代码块、引用的 Prompt、对话示例中，不视为正文语气问题。
- 若是用户原文中故意保留的对话示例，不应擅自删除，只需与正文区分清楚。

## 三、推荐替换方向

- “先说结论” → “定位概括” / “结论上”
- “如果只记一句话” → “概括而言”
- “可以理解为” → “可视为” / “可类比为”
- “通俗一点” → 直接改写为客观解释
- “我们要” → “需要”
- “你需要” → “需要”
- “学会” → “掌握”
- “快速搭建” → “完成基础搭建” / “构建原型”

## 四、句式要求

优先使用：

- 客观陈述句
- 明确判断句
- 条件句
- 分项说明句

谨慎使用：

- 感叹句
- 反问句
- 口语化插入语
- 过强主观评价

## 五、结构性风格要求

1. 同类小节的标签应尽量统一。
2. 术语首次出现时可带英文或缩写，后续保持一致。
3. 长段落可拆分，但避免拆成过碎的单句段。
4. 列表项应保持语法平行。
