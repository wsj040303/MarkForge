# Templates

以下模板用于 `Level 2` 与 `Level 3` 的章节整编。只有在原小节结构明显失衡时才使用，不得机械套用。

## 1. 知识讲解类

适用于概念、方法、技巧、协议说明等小节。

- 定义
- 解决问题
- 适用场景
- 示例
- 实现示例 / 实现思路

适用场景：

- RAG 技巧
- 检索策略
- 智能体方法论
- 协议与框架概念说明

## 2. 配置说明类

适用于配置文件、目录结构、工具接入方式说明。

- 作用
- 关键字段 / 关键配置
- 最小示例
- 使用建议 / 注意事项

适用场景：

- `.mcp.json`
- `settings.json`
- 配置目录说明

## 3. 选型比较类

适用于工具、框架、平台、收费方式的比较。

- 定位概括
- 核心差异
- 适用场景
- 选择建议

适用场景：

- 平台选型
- 框架选型
- 订阅制与按量计费比较

## 4. 协议 / 架构类

适用于通信协议、系统架构、组件说明。

- 定义
- 核心组件
- 工作流程 / 示例
- 局限性 / 边界

适用场景：

- MCP
- A2A
- Agent 架构

## 5. 经验判断类

适用于“如何选择”“何时使用”“常见标准”等小节。

- 判断目标
- 常见标准
- 适用条件
- 风险提示

适用场景：

- 获取 MCP
- Token 收费方式选择
- 模型配置切换时机

## 使用原则

1. 原文已清晰时，不强行模板化。
2. 原文仅轻微失衡时，优先微调，不整体重构。
3. 同一章内若多个小节使用同一种模板，应保持标签顺序一致。
4. 模板服务于理解，不服务于形式统一本身。
