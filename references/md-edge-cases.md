# Markdown Edge Cases

本文件用于处理 Markdown 文档中的高风险结构，避免误改。

## 1. 标题编号

若原文存在如下形式，应默认视为作者显式结构：

- `# 2`
- `## 2.1`
- `### 2.1.1`
- `#### 8.5.3`

处理规则：

1. 不得擅自删除编号。
2. 不得将显式编号标题改成无编号标题。
3. 若编号明显错误，可修正编号错误，但不得改变编号风格。

## 2. 代码块

代码块中的内容默认不参与正文结构判断。

包括但不限于：

- `#` 开头的代码注释
- Prompt 模板
- JSON
- YAML frontmatter
- Shell 命令
- Python / JavaScript 示例

处理规则：

1. 不将代码块内部的 `#` 识别为正文标题。
2. 不将代码块中的第一人称或口语当作正文语气问题。
3. 除非用户明确要求，否则不改代码块内容本身。
4. 若代码块围栏损坏，可修复围栏。

## 3. 表格

Markdown 表格常见风险：

- 对齐线损坏
- 列数不一致
- 段落重写后破坏表格结构

处理规则：

1. `Level 1` 可修表格格式，但不改表格内容。
2. `Level 2` 可将明显难读的表格改写成列表，但前提是原意不变且更清晰。
3. `Level 3` 才允许压缩冗余表格或将资料型表格转为教学型说明。

## 4. 引用块

引用块中常出现：

- 官方文档摘录
- 说明性备注
- 提示语

处理规则：

1. 先判断引用块是正文说明、外部引用还是示例。
2. 不得把引用块中的示例语气误改为正文语气。
3. 若引用块本身是作者说明，可按整体文风统一语气。

## 5. 图片与链接

处理规则：

1. 保留原图片位置，除非用户明确要求调整布局。
2. 不主动删除外链图片。
3. 修正文案时不得破坏链接目标。

## 6. 强度边界提醒

### Level 1

- 可修格式，不可改结构
- 可修病句，不可重构内容

### Level 2

- 可整编结构，不可主动扩写知识点
- 可统一模板，不可重写作者观点

### Level 3

- 可补充与压缩内容
- 仍需保留章节核心原意与用途
